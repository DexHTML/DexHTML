## Главные проекты

### TechRepair

**Статус:** private / internal project  
**Тип:** система учёта техники, QR-кодов, ремонтов, актов, справочников и аудита

TechRepair — мой самый серьёзный проект. Это внутренняя система для учёта техники и сервисных процессов.

Что заложено в проекте:

- карточки техники;
- QR-коды;
- история ремонтов;
- справочники;
- акты передачи;
- подменный фонд;
- отчёты;
- аудит действий;
- подготовка к API и интеграции с другими модулями.

Главная идея проекта:

> Техника должна иметь понятную карточку, историю, статус и быстрый доступ через QR-код.

TechRepair рассматривается не как отдельная страница, а как часть будущей сервисной экосистемы.

### ServiceHub / FairDesk

**Статус:** active development / private project  
**Тип:** заявочная система и экосистема внутренних сервисов

ServiceHub — идея единого рабочего центра, который объединяет несколько внутренних инструментов.

FairDesk — заявочная система внутри этой экосистемы.

Главная задача FairDesk:

> Пользователь просто сообщает проблему, а система помогает правильно направить заявку, учесть исполнителей, нагрузку, доступность и объяснить назначение.

В проекте прорабатываются:

- создание заявок;
- роли заявителя, исполнителя и администратора;
- связь с техникой;
- связь с графиком дежурств;
- автоматическое и ручное назначение;
- справедливое распределение нагрузки;
- статусы и архив;
- история действий;
- объяснение, почему заявка назначена конкретному исполнителю.

Главный принцип:

> Просто для людей. Умно внутри. Готово к развитию.

Проект важен не только как календарь, а как источник данных о доступности людей для будущей экосистемы.
